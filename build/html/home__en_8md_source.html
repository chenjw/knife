4 # Summary
5 
6 Knife is a command-line java runtime diagnostic tool like [Btrace](http://kenai.com/projects/btrace/). And provides many features btrace not support.
7 
8 # Features
9 
10 Similarities with btrace:
11 
12 1. Support runtime attach to the jvm.
13 2. Can trace a method invocation. And print out the arguments and result.
14 3. Can record the time-consuming of any invocation. And print it out.
15 
16 Features btrace not support:
17 
18 1. A command line interactive form. Without having to write the scripts.
19 2. Can find any object in the heap.
20 3. Can view any fields in an object. And change the field values.
21 4. Can invoke and trace any method of an object.
22 5. Can find any reference to an object.

41 # Commands
42 
43 ## find
44 
45 This command is used to find loaded classes in jvm.
46 
47 And can also find instances of a class in the heap by the class name.
48 
49 ~~~~~~~~~~
50 find <find-expretion>
51 ~~~~~~~~~~
52 
53  <h3>find-expretion</h3>
54 
55 className or class index.
56 
57 > find classes whose name like '%Service%'
58 
59 ![Find Service](find_service.jpg)
60 
61 > find instances of class 'com.chenjw.knife.server.test.impl.ApplyServiceImpl' in the heap.
62 > can also short for 'find 15' , '15' means the index of the last find result.
63 
64 ![Find Target Service](find_target_service.jpg)
65 
66 ## cd
67 
68 Mark an object as the target object. Some commands like 'invoke' and 'ls' will apply to the target object.
69 
70 Anywhere an object appears with a mark '@num' (maybe the result of 'find', or the param of 'invoke'), it can be mark as the target object using the command 'cd'.
71 
72 ~~~~~~~~~~
73 cd <object-id>
74 ~~~~~~~~~~
75 
76 > Into the instance of class 'com.chenjw.knife.server.test.impl.ApplyServiceImpl'

81 ## ls
82 
83 This command is used to list the fields of the target object, or to list the methods of the target object.
84 
85 ~~~~~~~~~~
86 ls [-f] [-m] [<classname>]
87 ~~~~~~~~~~
88 
89  <h3>classname</h3>
90 
91 Set <classname> to find static fields or methods , if <classname> not set , will apply to target object.
92 
93  <h3>-f</h3>
94 
95 List fields of target object/class, including static , instance fields, and the fields defined in superclass.
96 
97  <h3>-m</h3>
98 
99 List methods of target object/class, including static , instance methods, and the methods defined in superclass.
100 
101 > list fields of target object. Include both static and instance fields
102 
103 ![ls -f](ls_f.jpg)
104 
105 > list methods of target object. Include both static and instance methods.
106 
107 ![attach](ls_m.jpg)

139 ## invoke
140 
141 This command is used to invoke a method. And trace the invocation info.
142 
143 ~~~~~~~~~~
144 invoke [-f <filter-expretion>] [-t] <invoke-expretion>
145 ~~~~~~~~~~
146 
147  <h3>invoke-expretion</h3>
148 
149 Input 'package.ClassName.method(param1,param2)' to invoke static method, or 'method(param1,param2)' to invoke the method of target object. The params support json format.
150 
151  <h3>-f <filter-expretion></h3>
152 
153 Set <filter-expretion> to filter the invocation output you dont care.
154 
155  <h3>-t</h3>
156 
157 Is need output the method trace of the invocation.
158 
159 > just invoke, not trace
160 
161 ![attach](invoke.jpg)
162 
163 > invoke and trace
164 
165 ![attach](invoke_t.jpg)
166 
167 > invoke and trace and filter the invocation name.
168 
169 ![attach](invoke_f_t.jpg)

171 ## trace
172 
173 This command is used to wait and trace an invocation in the jvm.
174 
175 ~~~~~~~~~~
176 trace [-f <filter-expretion>] [-n <trace-num>] [-t] <trace-expretion>
177 ~~~~~~~~~~
178 
179  <h3>trace-expretion</h3>
180 
181 Input 'package.ClassName.method' to trace static method, or 'method' to trace the method of target object.
182 
183  <h3>-f <filter-expretion></h3>
184 
185 Set <filter-expretion> to filter the invocation output you dont care.
186 
187  <h3>-t</h3>
188 
189 Is need output the method trace of the invocation.
190 
191  <h3>-n <trace-num></h3>
192 
193 Trace times.
